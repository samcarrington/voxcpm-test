Method Chain:
  VoxCPM.generate_streaming()
    ↓
  VoxCPM._generate(streaming=True)
    ↓

  VoxCPMModel._inference(streaming=True)

  - Real-time factor: 0.5-2x (generates 80ms in 100-500ms)

Framework: FastAPI + WebSocket (recommended for low latency)

Architecture:
  Client (WebSocket)
    ↓
  FastAPI Server

    ├─ Accept WebSocket connections
    └─ Stream chunks via binary messages

        ↓

Key Implementation Points:
  1. Load model once at server startup
  2. Iterate over generate_streaming() generator
  3. Convert float32 chunks to int16 PCM or WAV
  4. Send via WebSocket for low latency
  5. Handle client disconnection gracefully

================================================================================
AUDIO CONVERSION OPTIONS
================================================================================

Option 1: Raw PCM (int16)
  - Simplest, lowest overhead
  - Convert: (chunk * 32767).astype(np.int16).tobytes()
  - Size: 2,560 bytes per chunk (1,280 samples × 2 bytes)

Option 2: WAV Format
  - Includes header, self-contained
  - Use: soundfile.write(buffer, chunk, 16000, format='WAV')
  - Size: ~2,600 bytes per chunk (with WAV header)

Option 3: Opus (Compressed)
  - Best for bandwidth-constrained scenarios
  - Use: opuslib.Encoder(16000, 1, APPLICATION_VOIP)
  - Size: ~200-400 bytes per chunk (8-16x compression)
